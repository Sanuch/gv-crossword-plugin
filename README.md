# Crossword Solver - Chrome Extension

Расширение для решения кроссвордов на сайтах Godville с поддержкой различных API-провайдеров.

## Новые возможности (Обновление)

### 🎨 Улучшенный интерфейс
- **Широкое popup окно** - popup теперь имеет ширину 600px для лучшего отображения информации
- **Модальное окно** - альтернативный полноэкранный режим для решения кроссвордов
- **Современный дизайн** - обновленные стили с градиентами и анимациями

### � Улучшенное отображение результатов
- **Индивидуальные карточки** - каждое слово отображается в отдельной карточке
- **Результаты под словами** - ответы показываются непосредственно под каждым словом
- **Цветовая индикация** - первый ответ выделен как основной
- **Счетчики ответов** - показывается количество найденных вариантов

### � Система кеширования
- **Автоматическое кеширование** - результаты сохраняются на 24 часа
- **Ключ кеширования** - использует значение из `div#date span` как уникальный ключ
- **Индикатор кеша** - показывает, когда данные загружены из кеша
- **Fallback ключ** - использует дату и домен, если элемент даты не найден

## Режимы работы

## Использование

1. Откройте страницу с кроссвордом на Godville
2. Нажмите на иконку расширения
3. Выберите провайдер API
4. Нажмите "🚀 Solve"
5. Результаты отобразятся под каждым словом
6. При повторном решении того же кроссворда данные загрузятся из кеша

## Кеширование

Система кеширования работает следующим образом:

1. **Ключ кеша**: `{домен}_{значение_из_div#date_span}`
2. **Время жизни**: 24 часа
3. **Хранилище**: Chrome Storage API (обычный режим) или localStorage (модальный режим)
4. **Fallback**: Если элемент даты не найден, используется текущая дата

### Структура объекта запроса
- `lang` - код языка кроссворда (`ru`, `en`, `de`, `fr`, и т.д.)
- `tasks` - массив слов для решения

### Структура объекта слова
- `id` - идентификатор слова (например: "2г" = 2-е слово по горизонтали, "3в" = 3-е слово по вертикали)
- `pattern` - шаблон слова, где `_` обозначает неизвестные буквы
- `clue` - подсказка из кроссворда
- `length` - длина слова в буквах
- `answers` - массив возможных ответов (только в ответе от API)

## 🔍 Как работает извлечение данных

1. **Поиск контейнера:** расширение ищет элемент с ID `twl_wrap`
2. **Анализ сетки:** парсит таблицу `cross_tbl` и определяет активные ячейки
3. **Определение языка:** анализирует заголовки подсказок и домен
4. **Извлечение номеров:** находит ячейки с номерами слов в элементах `.num`
5. **Формирование шаблонов:** создает паттерны слов, заменяя неизвестные буквы на `_`
6. **Парсинг подсказок:** извлекает подсказки из элемента `.cross_q`
7. **Сопоставление:** связывает слова с подсказками по номерам и направлениям

## 🎯 Заполнение ответов

1. **Поиск начальных ячеек:** находит ячейки с соответствующими номерами
2. **Определение направления:** горизонтальное (г) или вертикальное (в)
3. **Заполнение по буквам:** находит input поля и вставляет буквы
4. **Триггер событий:** вызывает события для сохранения в localStorage сайта

## 🛠️ Разработка

Для разработки собственного API сервиса:

1. Принимайте POST запросы с объектом, содержащим `lang` и массив слов
2. Используйте параметр `lang` для выбора соответствующего словаря
3. Возвращайте массив слов, дополненный полем `answers`
4. Поддерживайте CORS для работы с расширением
5. Обрабатывайте ошибки и возвращайте соответствующие HTTP коды

### Пример обработки языка в API:
```javascript
function processRequest(requestData) {
  const lang = requestData.lang || 'ru';
  const dictionary = getDictionary(lang);
  
  return requestData.tasks.map(word => {
    const answers = findAnswers(word.pattern, word.clue, lang, dictionary);
    return { ...word, answers };
  });
}
```
